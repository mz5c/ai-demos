# SSE 流式作文阅读器 — 设计文档

## 概述

一个基于 Python FastAPI + SSE（Server-Sent Events）的流式作文阅读页面。用户打开页面，点击按钮后以打字机效果流式输出作文内容，速度可调。

## 架构

```
┌──────────────┐     SSE /api/stream     ┌──────────────────┐
│  浏览器页面   │ ◄────────────────────── │  FastAPI Server  │
│  (HTML+JS)   │                         │  (Python + uvicorn)│
└──────────────┘                         └──────┬───────────┘
                                                 │
                                         ┌───────▼───────────┐
                                         │  全国II卷高考作文.md │
                                         │   (源文件)          │
                                         └───────────────────┘
```

### 后端（FastAPI + uvicorn）

- **单文件** `server.py`
- 端口：`8080`（可配置）
- 接口：
  - `GET /` — 返回 index.html 页面
  - `GET /api/stream?speed={ms}` — SSE 流式接口，按指定间隔（毫秒）逐字推送作文内容
  - `GET /api/info` — 返回作文基本信息（标题、总字数）

### SSE 协议

```
event: start
data: {"total": 1024, "title": "守其体者终复明"}

event: char
data: {"char": "日", "index": 0, "done": false}

event: char
data: {"char": "月", "index": 1, "done": false}

...

event: done
data: {"index": 1024, "done": true}
```

### 前端（单页 HTML + CSS + JS）

- **设计风格**：编辑/文艺风（暖色调、衬线字体、纸质书质感）
- **EventSource** 接收 SSE 事件
- **速度控制**：滑块，对应每个字的间隔（30ms ~ 200ms），默认为 60ms/字
- **状态控制**：开始按钮，首次点击开始；完成后可重新开始
- **渲染**：逐字追加到内容区，带闪烁光标
- **进度**：底部进度条 + 字数统计

## 速度控制映射

| 滑块位置 | 间隔 | 效果 |
|---------|------|------|
| 慢 | 150-200ms | 逐字缓慢出现 |
| 中（默认） | 60-80ms | 流畅打字机效果 |
| 快 | 30-40ms | 快速填充 |

## 边界情况处理

- 文件不存在或为空 → SSE 返回 error 事件，页面显示提示
- 中途断开连接 → 页面提示连接断开，按钮变为重试
- 速度滑块在流式过程中可实时调整
- 完成后重置按钮，可再次从头播放

## 测试

- 确认 SSE 接口返回正确的 event stream 格式
- 确认不同速度参数生效
- 确认页面按钮状态流转正确（初始 → 播放中 → 结束）
